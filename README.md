# CS 210 Portfolio Reflection

## Project Summary

For my CS 210 portfolio artifact, I selected my Chada Tech Clocks project. This project was designed to display both a 12-hour clock and a 24-hour clock side by side. The program allowed the user to add one hour, one minute, or one second to the current time using a menu. The main problem this project solved was creating a functional time display that updates correctly and shows time in two different formats.

## What I Did Well

One thing I did particularly well was organizing the program into separate functions. Instead of putting all the code into one large main function, I used functions for adding hours, minutes, and seconds, displaying the clocks, and showing the menu. This made the code easier to read, test, and understand. I also did well formatting the clock output so the time displayed clearly.

## Where I Could Enhance the Code

One area where I could improve the code is input validation. I could make the program better at handling incorrect input, such as letters or numbers outside the menu options. This would make the program more secure and reliable because it would prevent unexpected behavior. I could also improve the program by allowing the user to enter a starting time instead of only using the time already set in the program.

## Challenges and How I Overcame Them

The most challenging part of this project was making sure the time rolled over correctly. For example, when seconds reached 60, the program needed to reset seconds to 0 and add one minute. The same idea applied to minutes and hours. I overcame this by breaking the problem into smaller functions and testing the program step by step. I used course materials, zyBooks examples, Visual Studio, and debugging as part of my support network.

## Transferable Skills

The skills from this project that will transfer to other projects include writing functions, using loops, handling user input, formatting output, and debugging code. These are important programming skills that I will use in future C++, Java, Python, and other programming courses. This project also helped me better understand how to design a program before writing all of the code.

## Maintainability, Readability, and Adaptability

I made the program maintainable, readable, and adaptable by using clear function names, comments, and organized code. Each function had a specific purpose, which makes the program easier to understand and update later. The program is adaptable because I could change one part, such as the clock display or menu options, without rewriting the entire program.
